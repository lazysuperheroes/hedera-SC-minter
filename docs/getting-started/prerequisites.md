# Prerequisites and Development Setup

This guide covers everything needed to build, test, and deploy contracts in the Hedera SC Minter suite.

## Requirements

- **Node.js 18+** (LTS recommended)
- **npm** (ships with Node.js)
- A **Hedera testnet account** with an ED25519 key pair (for running tests and deployments)

### Obtaining a Hedera Testnet Account

1. Visit the [Hedera Portal](https://portal.hedera.com/) and create a testnet account.
2. Note your **Account ID** (e.g., `0.0.12345`) and **Private Key** (ED25519 hex).
3. The portal provides initial testnet HBAR for development.

## Installation

```bash
git clone https://github.com/Burstall/hedera-SC-minter.git
cd hedera-SC-minter
npm install
```

## Environment Configuration

Copy the example environment file and fill in your credentials:

```bash
cp .env.example .env
```

### Required Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `ACCOUNT_ID` | Hedera operator account (ED25519) | `0.0.12345` |
| `PRIVATE_KEY` | Operator private key (hex) | `302e...` |
| `ENVIRONMENT` | `TEST` for testnet, `MAIN` for mainnet | `TEST` |

### Optional Variables (per contract)

| Variable | Used By | Description |
|----------|---------|-------------|
| `CONTRACT_NAME` | MinterContract / SoulboundMinter scripts | Active contract name |
| `LAZY_TOKEN_ID` | All $LAZY-enabled contracts | $LAZY token address (differs per network) |
| `LAZY_SCT_CONTRACT_ID` | All $LAZY-enabled contracts | Lazy Smart Contract Treasury address |
| `FOREVER_MINTER_CONTRACT_ID` | ForeverMinter scripts | Deployed ForeverMinter address |
| `NFT_TOKEN_ID` | ForeverMinter scripts | NFT token to distribute |
| `PRNG_CONTRACT_ID` | ForeverMinter, EditionWithPrize | PRNG generator address |
| `LAZY_GAS_STATION_CONTRACT_ID` | ForeverMinter | LazyGasStation address |
| `LAZY_DELEGATE_REGISTRY_CONTRACT_ID` | ForeverMinter, EditionWithPrize | Delegate registry address |
| `EDITION_WITH_PRIZE_CONTRACT_ID` | EditionWithPrize scripts | Deployed EditionWithPrize address |

See `.env.example` for the complete list with testnet/mainnet defaults.

## Compiling Contracts

```bash
npx hardhat compile
```

This also runs `hardhat-contract-sizer` to report deployed sizes. All contracts must stay under the 24,576-byte limit.

**Compiler settings:** Solidity 0.8.18, optimizer enabled (200 runs), `viaIR: true`.

## Running Tests

Tests run against **Hedera testnet** (not a local Hardhat network). They create real accounts, tokens, and contracts on testnet, so a funded testnet account is required.

```bash
# All tests
npm test

# Per-contract test suites
npm run test-ft          # FungibleTokenCreator
npm run test-nft         # MinterContract
npm run test-sbt         # SoulboundMinter
npm run test-badges      # SoulboundBadgeMinter
npm run test-forever     # ForeverMinter
npm run test-ewp         # EditionWithPrize

# Single test file
npx hardhat test test/ForeverMinter.test.js
```

## Linting

```bash
# Solidity linting
npm run solhint
```

JavaScript follows ESLint (`eslint:recommended`) with tabs, single quotes, semicolons, and `stroustrup` brace style.

## Deploying Contracts

Deployment scripts are interactive (using `readline-sync`):

```bash
npm run deploy-ft                                             # FungibleTokenCreator
npm run deploy-nft                                            # MinterContract
npm run deploy-sbt                                            # SoulboundMinter
npx hardhat run scripts/deployment/deploy-ForeverMinter.js    # ForeverMinter
npx hardhat run scripts/deployment/deploy-EditionWithPrize.js # EditionWithPrize
```

## Project Structure (Key Directories)

| Path | Contents |
|------|----------|
| `contracts/` | Solidity source files |
| `test/` | Hardhat test suites (run on testnet) |
| `scripts/deployment/` | Deployment scripts |
| `scripts/interactions/` | CLI admin and user scripts per contract |
| `scripts/debug/` | Error decoding, contract info, log retrieval |
| `abi/` | Extracted ABIs (generated by `scripts/deployment/extractABI.js`) |
| `utils/` | Test/script helpers: `solidityHelpers.js`, `hederaHelpers.js`, `hederaMirrorHelpers.js`, `transactionHelpers.js`, `gasHelpers.js` |
| `docs/` | Design documents, business logic, testing plans |
